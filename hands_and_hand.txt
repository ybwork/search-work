Важно: главное не срок, за который будет сделано тестовое, а его качество. Уделите этому особенное внимание.
Задача:
Используя php-фреймворк Yii (возможно использование laravel) создать веб-приложение. 
В приложении должна быть CRUD-админка пользователей, у пользователя есть только логин и пароль. 
Приложение по расписанию (можно реализовать управление расписанием в админке) должно забирать данные о погоде любого (хотя можно в админке реализовать список городов) города с сервиса https://openweathermap.org/api и хранить его в локальной БД (MySQL или PostgreSQL)
Реализовать API со следующими end-point’ами:
1. POST /auth {login, password} -> { token }
Запрос авторизации. На входе в теле запроса принимает JSON с логином и паролем. На выходе возвращает временный токен доступа; 
1. GET /weather  -> { city, current_temp }
Запрос получения данных. В заголовке запроса проверять токен в поле token, и если токен валидный то возвращает JSON со списком городов (если их много) или с одним именем города и текущей температурой в них.
Сделать тестовую страничку для проверки end-point’ов


Обязательные требования:
* Результат работы надо упаковать в docker-контейнеры (приложение отдельно, бд отдельно) которая должны запускаться docker-composer’ом
* Исходные файлы со всей историей commit’ов залить в открытый репозиторий на GitHub
